Form Manual

- A technical description of elements to be used when creating Web forms. This covers elements that are not already described in the Web manual.

Please note: This is the least polished part of the .ny design. It will be worked on during October and November by those projects that have a need for form elements.

How to use this

This page only demonstrates form functionality. General specifications for typography, colors, icons and so on are found in the Web manual. This includes elements like buttons and modals, because they are not only form elements, but also navigation elements.

The form design elements is available from a shared depository on Amazon. You can link to the files from there on an https://elements.storebrand.no URL.

Forms: Text input fields

(Not finished, but you can start using it anyway. Please consult storebrand.ny project.)

Forms are built using namespaced bootstrap-classes, stb-form-inline, stb-form-group, and so on. We have not over-ridden the standard Bootstrap focus indicator (a surrounding faint color gradient), because by now that is universally recognized as helpful to the general public.

Code: 
<form>
  <h5>Pick your poison</h5>
    <!-- Radio buttons -->
    <input class="stb-radio" type="radio" id="a" value="a" name="aorb" />
    <label for="a">I want poison A</label>
    <input class="stb-radio" type="radio" id="b" value="b" name="aorb" checked />
    <label for="b">I want poison B</label>
    <!-- Checkboxes -->
    <input class="stb-checkbox" id="checkbox1" type="checkbox" />
    <label for="checkbox1">Check this out</label><br>
    <input class="stb-checkbox" id="checkbox2" type="checkbox" />
    <label for="checkbox2">Check that out</label><br>
    <input class="stb-checkbox" id="checkbox3" type="checkbox" disabled />
    <label for="checkbox3">Can't touch this (disabled)</label>
</form>

Be precise when using input+label; ID and for-attributes must match, and the order must be input, then label.

Dropdown / Select

(Not finished, but you can start using it anyway. Please consult storebrand.ny project.)

Increasingly, the best choice is to use native form elements. For advanced form elements, this is doubly so.

Consider what happens when your form is presented on a smart phone, a "smart-TV" or, theoretically, a Windows Phone. Manufacturers of these devices put a lot of work into creating the best possible user experience. Customizing form elements will generally obstruct that goal.

Recommended default BS-style select

<select class="form-control">
  <option value="a">Select A</option>
  ...
</select>

Custom style with autocomplete

Note: Type-ahead requires chosen.jquery.js + stb-chosen-less.

<select data-widget="chosenSelect" class="form-control" data-placeholder="Pick a number">
  <option value=""></option>
  <option value="1">Number one</option>
  <option value="2">Number two</option>
  <option value="3">Number three</option>
  <option value="4">Number four</option>
  <option value="4">Number five</option>
</select>
<script>
  $('select[data-widget=chosenSelect]').chosen();
</script>

Sliders

(Not finished, but you can start using it anyway. Please consult storebrand.ny project.)

As we wait for browsers world-wide to deliver excellent sliders through the HTML5 range tag, we currently use a customized slider.

A slider must always be accompanied by a good old text input field, for universal design.


Note: Our slider is based on Egorkhmelev's jslider (see GitHubfor code with more options), stripped down and with a few improvements:

  • Self-contained JS-file, all dependencies are included at the top of the file.
  • Improvements & some pull-requests have been included (resize, redraw-functions ++)
  • Minor edits to the markup template
  • Customized styles/graphics

Required JS + CSS:

  • jquery.slider.all.js
  • stb-ui-slider.less

Progress bar

(Not ready to use, but if you need something like this, consult with storebrand.ny project.)

For use cases where you can show a progress bar based on percentage (0-100), and not necessarily on a set number of steps, you can use this progress bar.

to see random progress jumps up and down the scales below:

60% Complete
60% Complete

Code: 

<div class="stb-progress"> <!-- add class .gray for gray version -->
  <div class="stb-progress-bar" data-widget="progressExample" role="progressbar"
    aria-valuenow="60" aria-valuemin="0" aria-valuemax="100" style="width: 60%;">
    <span class="sr-only">60% Complete</span>
  </div>
</div>
<script>
  $('div.stb-progress div[data-widget=progressExample]').css({width: newPercentValue + '%'});
</script>

The .sr-only class is used as a fallback for screen readers. Animation is done through css transitions.
