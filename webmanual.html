Web Manual

- A technical description of elements to be used when creating Web content.

How to use this

Underneath each section on this page, we indicate how ready that section is for use by any project. Contact the storebrand.ny project for clarifications or requests for adding new elements. As of late August 2014 we still make minor visual adjustments, but the tag mark-up should remain unchanged. New elements will be added as needed.

The design will be made available from a shared depository, probably on Amazon. The URL to use is not yet available. If you have any cross-site scripting issues, for now you must copy the items you need to your own project. We will establish the Amazon files as a .storebrand.no domain with SSL before launch of storebrand.ny, but it's not there yet.

Elements for user interaction should log activity to Google Analytics. This is in some cases not yet implemented. JavaScript for this will be available later.

Grid and breakpoints

Grid: Based on 12 columns from Bootstrap 3. We can responsively divide the contents horizontally into any combination of 12 in total (12, 6+6, 3+3+3+3, 4+4+4, and so on).

Our screen width breakpoints are:
Extra small = up to 767px, Small = 768-991px, Medium = 992-1199px, Large = 1200px and up.

Typography / Text

(Safe to use)

Lead paragraph

Lead paragraph (intro): This is bolder. This is where the author grabs the attention of the reader. Lead paragraphs usually include brief answers to all main questions answered by the following text when it comes to who, what, why, when, where, and how.

Code: <p class="intro">...</>
font-size: 24px; line-height: 30px; font-weight: 700;

General paragraph

A paragraph is a passage in a section of text that is about a different subject from the preceding text section, marked by commencing on a new line.

And then the next paragraph follows. It's about something else. The line-height may seem a bit excessive, but it's needed to create sufficient space below characters for the link indicator line, see above.

Code: <p>...</p>
font-size: 16px; line-height: 24px;

Small print

In some cases, like for publishing dates, source references and footnotes, you can use small, like so:
This part of the text is small and not very important, but still possible to read.


Code: <small> or <span/p/div class="petit">
font-size: 12px; line-height: 16px;

Blockquote

Layout / Spacing

(Safe to use)

We have a number of classes that can be used on any element to provide some spacing in different ways. Depending on your situation, you might of course want to just use straight CSS styling instead, but these classes come in handy when we want to let our non-IT Web editors use our CMS to work on contents and space things out.

(Note: The actual margins/paddings are NOT 5, 10, ... or 50. That's just a simplification to help people remember the classes. Right now, the numbers are mapped to pixels in the following way: 5 to 6, 10 to 12, 20 to 24, 30 to 30, 40 to 36 and 50 to 48. Sorry about the confusion, if any.)

Vertical spacing

The classes give 10, 20, 30, 40 or 50 pixels and are used this way:

class="top-margin-20" => There's a margin-top of 20 pixels added to the element.
class="top-margin-30-mdlg" => A margin-top of 30 pixels is added, but only on md and lg width devices. Devices with xs or sm width gets no margin-top.

The same goes for bottom-margin-10 through bottom-margin-50.

You can use the same pattern for padding, e.g. class="top-padding-50" gives padding-top of 50 to that element, and you can use the -mdlg extension to make it valid only for md and lg width devices.

Spacing on the sides

You can add margins and padding on the sides in the same way, and if desired for large and medium screen widths only, using these classes: left-margin-x and left-margin-x-mdlg, where x can be 5, 10 or 20. And you can of course do the same with right-margin, left-padding and right-padding.

Usage of width

There are also classes for setting the width of an element that otherwise would inherit the full width from its parent element. They were introduced in order to prevent clashes between background graphics pulled to the right when contents pulled to the left needs to be responsive.

There are only two of these classes; class="width-50" and class="width-75", giving widths of 50% and 75%, and you can add the -mdlg extension to let the rule only be valid for md and lg width devices. Devices with sm and xs widths will then keep their original width, usually 100%.

Border use

And then there's the class "border-top-and-bottom", which adds visible borders at the top and bottom of the element.

Tables

(Ready to use)

Tables shall not be used to space out contents, but in some cases it is a good way to present a list. Although we can use a hovereffect on rows, remember that this doesn't work on touch devices.

For a wide table to be responsive, you may have to create two tables with different visibility and different content for wide and narrow screens, or you can show/hide columns in a wide table.

Tables have no color varieties, but will always follow the styles displayed below.

Tables have no vertical borders anywhere.

Table font-size is 16px by default. By adding the class "small-text" to the table tag, the font-size is reduced to 14.

FAQ

(Ready to use)

Frequently Asked Questions ("Accordion")

The FAQ "accordion" can be used to initially hide substantial contents on pages that might otherwise seem overwhelming at first glance. (For use only in cases where it's really not possible to reduce the amount of text, of course.)

Example:


Alt om din pensjon

  • Hvordan får jeg oversikt over hva jeg får i pensjon? 1
    • Har du pensjon i Storebrand, logg inn på dine personlige sider for fullstendig oversikt. Der kan du også hente inn tallene dine fra folketrygden (NAV).
    • Har du fripoliser eller pensjonskapitalbevis fra tidligere arbeidsgivere, finner du disse på minpensjon.no
    • Prøv vår kalkulator som enkelt viser deg hva du kan forvente å få i pensjon. Vi gir deg også en vurdering av hvor mye du bør spare for å få den pensjonen du ønsker deg.
  • Hva består egentlig pensjon av? 2
    Pensjon består av:
    1. Folketrygden - opptjening betalt av staten. Hvor mye du får, kommer an på hvor gammel du er, ditt lønnsnivå og hvor lenge du har jobbet.
    2. Tjenestepensjon - dette betales av din arbeidsgiver. Her finnes det forskjellige avtaler og det lønner seg å være bevisst på hva du får fra din arbeidsgiver.
    3. AFP - Avtalefestet pensjon. Betales av arbeidsgiver, men fungerer litt som folketrygden. Ikke alle har denne avtalen, så undersøk hos din arbeidsgiver.


The FAQ is contained in a normal ul list, but needs a data-widget attribute set to "stbFaq": 

<ul class="accordion" data-widget="stbFaq">
  <li>
    <a href="#" class="twister">Question here
      <span class="number">Optional numbering<span>
      <span class="expander"><span>
    </a>
    <div class="answer"> Answer-text here (a,b,i,ul,ol allowed) </span>
  </li>
  <!-- One more li item per question/answer pair -->
</ul>

<script>
// Initialize the padding of the FAQ with necessary tagging:
$('ul[data-widget=stbFaq]').stbFaq();
</script>

Icons

(Ready to use. More icons are coming.)

We have two sets of icons; one with a few icons with colors that are fixed and another with some other, more general icons available in the colors charcoal, white and red, and in the sizes 16, 24, 32 and 48 pixels width and height.

Note! The icons come in only two sizes, small and medium. On standard screens we use 16 and 24 pixel size icons. On retina screens we use 24 and 48 pixel size icons. This has NOT been implemented in the examples below yet. A full retina-fication will be done eventually. Class "stb-sprite-16" will then be replaced with classES "stb-sprite small", "stb-sprite-24" will be "stb-sprite medium", "stb-sprite-32" will be "stb-sprite small" with a media query to adjust for size and "stb-sprite-48" will be "stb-sprite medium" with a media query to adjust for size.

</form>

Be precise when using input+label; ID and for-attributes must match, and the order must be input, then label.

Dropdown / Select

(Not finished, but you can start using it anyway. Please consult storebrand.ny project.)

Increasingly, the best choice is to use native form elements. For advanced form elements, this is doubly so.

Consider what happens when your form is presented on a smart phone, a "smart-TV" or, theoretically, a Windows Phone. Manufacturers of these devices put a lot of work into creating the best possible user experience. Customizing form elements will generally obstruct that goal.

Recommended default BS-style select

<select class="form-control">
  <option value="a">Select A</option>
  ...
</select>

Custom style with autocomplete

Note: Type-ahead requires chosen.jquery.js + stb-chosen-less.

<select data-widget="chosenSelect" class="form-control" data-placeholder="Pick a number">
  <option value=""></option>
  <option value="1">Number one</option>
  <option value="2">Number two</option>
  <option value="3">Number three</option>
  <option value="4">Number four</option>
  <option value="4">Number five</option>
</select>
<script>
  $('select[data-widget=chosenSelect]').chosen();
</script>

Progress bar

(Not ready to use, but if you need something like this, consult with storebrand.ny project.)

For use cases where you can show a progress bar based on percentage (0-100), and not necessarily on a set number of steps, you can use this progress bar.

to see random progress jumps up and down the scales below:

60% Complete
60% Complete

Code: 

<div class="stb-progress"> <!-- add class .grey for grey variant -->
  <div class="stb-progress-bar" data-widget="progressExample" role="progressbar"
    aria-valuenow="60" aria-valuemin="0" aria-valuemax="100" style="width: 60%;">
    <span class="sr-only">60% Complete</span>
  </div>
</div>
<script>
  $('div.stb-progress div[data-widget=progressExample]').css({width: newPercentValue + '%'});
</script>

The .sr-only class is used as a fallback for screen readers. Animation is done through css transitions.
